# Contributing to this Project

Thank you for taking the time to contribute to this project! There are several
ways you can contribute:

## Creating an Issue

If you spotted a bug, want to propose a new feature, or want to discuss an
enhancement, please create a new issue! 

* Issues must have clear and descriptive titles.
* Issues must be concrete about the problem, and/or the solution that you
  propose.
* If the issue is a bug report, 
    1. Describe the context, and the exact steps that reproduce the problem.
       (include screenshots or animated GIFs when needed)
    2. Clearly describe the behavior observed, and the behavior you expected.

## Submitting a Pull Request

Once you have a issue you would like to work on, and/or have some changes to
introduce to the repo, please follow these steps:

1. Fork this repository, 
2. Create a new branch,
3. Add your changes,
4. And submit a pull request!

Things to note when creating a pull request:

* Provide a high-level explanation of what your changes do.
* Commit often, with descriptive commit messages overviewing the changes you made.
* Add tests that test both the success and failure cases of the relevant code.
* When the upstream `main` branch becomes out of sync, rebase your feature
  branch on `main`.
* The code style must follow the official [PEP 8 Style
  Guide.](https://peps.python.org/pep-0008/)
